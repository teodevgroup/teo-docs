# Serve

The `serve` command performs necessary database 
[migration](/concepts/components/teo-cli/migration)
<span className="concept">CONCEPT</span>, 
[seeding records](/concepts/components/teo-cli/seed)
<span className="concept">CONCEPT</span>, and starts the HTTP server.

## Start the server

Use `serve` command to start the server.

<RustDoc>
```sh
cargo teo serve
```
</RustDoc>
<NodeJSDoc>
```sh
npx teo serve
```
</NodeJSDoc>
<PythonDoc>
```sh
teo serve
```
</PythonDoc>

## Skip the migration

To skip the auto migration, pass the option `--no-migration`.

<RustDoc>
```sh
cargo teo serve --no-migration
```
</RustDoc>
<NodeJSDoc>
```sh
npx teo serve --no-migration
```
</NodeJSDoc>
<PythonDoc>
```sh
teo serve --no-migration
```
</PythonDoc>

## Skip the seeding

To skip the auto seeding, pass the option `--no-autoseed`.

<RustDoc>
```sh
cargo teo serve --no-autoseed
```
</RustDoc>
<NodeJSDoc>
```sh
npx teo serve --no-autoseed
```
</NodeJSDoc>
<PythonDoc>
```sh
teo serve --no-autoseed
```
</PythonDoc>
